# Clean code: 
  - Legibilidade:  um código de fácil leitura
  - Manutenível: Código de fácil manutenção
    - Se uma pessoa consegue ler o código, mas nao consegue dar manutenção, o código não é limpo.


# Precisamos escrever muito código ruim pra começar a escrever clean code
  - clean code é uma prática, nao é aplicado do dia pra noite

# Clean code NÃO É:
  1. Manual. Clean code é prática
  2. Arquitetura de pastas (isso é clean architecture)
  3. Um código menor
  4. Arquitetura
  5. Performance - seguir o clean code nao vai tornar a aplicacao mais performática

# Clean code É:
  1. Previsibilidade
  2. Confiança
  3. Manutenibilidade  
  4. Legibilidade
